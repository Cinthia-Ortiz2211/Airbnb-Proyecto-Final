# Convención de Commits

## Fuente principal
- Conventional Commits 1.0.0 (https://www.conventionalcommits.org/en/v1.0.0/)

## Descripción
La convención define un formato estandarizado para los mensajes de commit con el objetivo de facilitar la generación automática de versiones (semver), changelogs y análisis de cambios.

## Estructura del mensaje

`<tipo>(<ámbito (opcional)>): <descripción corta>`

### Tipos comunes
- `feat`: nueva funcionalidad.
- `fix`: corrección de error.
- `chore`: tareas menores o de mantenimiento.
- `docs`: cambios en documentación.
- `refactor`: mejoras internas sin cambiar funcionalidad.
- `test`: cambios o adición de pruebas.

## Ejemplo

- `feat: agregar modelo de usuario`
- `fix: validar inicio de sesion`
- `docs: agregar diagrama uml`
